1.https://github.com/saydals/bf  여기가 2.md 디렉토리 파일들의 원격 저장소다.

2. 커밋을 명령하면 푸시와 웹게시용 파일 업로드는 자동으로 한다.

3. 사용자가 주문한 디렉토리 (2.md파일이 존재하는 디렉토리 - 하위포함)내에서만 작업한다.

4. 사용자의 요구 사항이 명확하지 않으면 추측하지 않고 사용자에게 묻는다.

5. apk 생성을 위한 도구들은 이미 모두 갖춰져 있다. 사용자가 별도로 빌드할 필요가 없다. 

6. 어떤 경우도 공식저장소 https://github.com/betaflight 로 복원하지 않는다. 필요시 참고만 한다.

   필요하지만 로컬저장소에 없는 경우에만 저장 가능. 

7. 원상 복구는 원격 저장소 https://github.com/saydals/bf 를 사용한다.

8. 사용자가 윈도우 상에서 파일 수정을 수시로 하기때문에 기록없는 파일 변경을 오류로 판단하지 않는다.

10. apk 생성 방법:

    - cd /home/betaflight/configurator
    - export ANDROID_HOME=/home/betaflight/android-sdk
    - export JAVA_HOME=/usr/lib/jvm/java-21-openjdk-amd64
    - npx vite build (웹 빌드)
    - npx cap sync android (Capacitor 동기화)
    - cd android && ./gradlew assembleDebug (디버그 APK)
      또는 ./gradlew assembleRelease (릴리즈 APK)
    - 생성 APK 경로:
        디버그: android/app/build/outputs/apk/debug/app-debug.apk
        릴리즈: android/app/build/outputs/apk/release/app-release-unsigned.apk

11. 디버그 APK(app-debug.apk)는 개발자 서명(debug keystore)으로 서명되어 있어 기기에 바로 설치(adb install)할 수 있다. 릴리즈 APK는 별도의 release 서명 키가 필요하다.

12. 첫 빌드 시 Android SDK가 필요하며(/home/betaflight/android-sdk), npm install, 공식 저장소에서 누락된 보조 파일(libraries/, src/js/msp/debug/ 등) 복사가 필요할 수 있다.
